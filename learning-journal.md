## 已暴露的典型认知偏差

### 1. HTTP 请求链路理解停留在框架内部

当前能想到的主要是：

- URL 路由
- Controller 分发
- 业务处理
- 响应返回

当前缺失的关键链路包括：

- DNS 解析
- TCP 建连
- IP + 端口寻址
- 服务端监听端口
- 内核协议栈把请求交给应用进程

### 2. TCP 三次握手理解不清

当前直觉是“双方互相说准备好了就行，两次看起来也够”。

后续需要重点补：

- 三次握手到底在确认什么
- 为什么不是两次
- 为什么要防止历史连接请求误判

### 3. 进程和线程概念混淆

当前错误表述：

- “进程是物理定死的通道数”

后续需要纠正为：

- 进程是资源分配的基本单位
- 线程是 CPU 调度的基本单位
- 线程共享进程资源

### 4. 阻塞 / 非阻塞 与 同步 / 异步混淆

当前倾向把“非阻塞”理解成“挂起后轮询结果”。

后续需要拆开理解：

- 阻塞 / 非阻塞：调用时当前线程会不会卡住
- 同步 / 异步：结果获取方式是谁主动推进
