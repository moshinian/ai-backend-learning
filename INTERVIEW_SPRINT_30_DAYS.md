# 30 DAY INTERVIEW SPRINT

## 1. 目标

这是未来 1 个月的面试冲刺计划。

它不是替代 `LEARNING_ROADMAP.md`，而是从长期路线图中抽出当前最该优先突破的内容。

时间假设：

1. 总周期：4 周
2. 每周投入：30 小时
3. 总投入：约 120 小时

核心目标：

1. 补齐 Java 后端面试最高频基础主线
2. 把真实项目经验转成可面试表达
3. 建立必要的算法保底能力
4. 通过模拟面试暴露并修正表达问题

不追求：

1. 一个月内彻底精通所有后端知识
2. 平均推进 Roadmap 里的全部 Phase
3. 只看笔记、不输出、不复述

---

## 2. 本月优先级

最高优先级：

1. MySQL：事务、MVCC、索引、锁
2. 请求链路：Tomcat、线程池、Servlet、Spring MVC
3. Java 并发：线程、线程池、锁、CAS、AQS 基础
4. Redis / MQ / 分布式：缓存、分布式锁、消息可靠性、幂等、最终一致性
5. 项目表达：云服务结算系统、RAG 知识问答系统
6. 算法保底：滑动窗口、二分、动态规划入门

暂缓深入：

1. JVM 高级调优
2. 复杂系统设计专题
3. 图论、复杂 DP、题海式刷题
4. AI Backend 过深的模型训练和算法原理

## 4. 每周计划

### Week 1：请求链路 + 事务主线

目标：

1. 完成网络请求链路第一轮闭环
2. 补齐 Tomcat、线程池、Servlet、Spring MVC 的关系
3. 建立数据库事务第一轮完整理解

重点任务：

1. Tomcat 为什么能同时处理很多请求
2. 连接、请求、线程、长连接之间的边界
3. Servlet 容器、`DispatcherServlet`、Controller 的调用关系
4. 事务为什么出现
5. ACID 分别解决什么问题
6. 脏读、不可重复读、幻读和隔离级别
7. MVCC 为什么出现

验收标准：

1. 能讲清一个请求如何进入 Controller
2. 能讲清慢请求为什么会拖垮 Tomcat 工作线程
3. 能讲清事务和隔离级别解决的真实问题

---

### Week 2：MySQL 打穿

目标：

1. 把 MySQL 变成可稳定输出的面试模块
2. 把事务、索引、锁和项目场景连起来

重点任务：

1. undo log、redo log、binlog 的作用边界
2. Read View 和 MVCC 基本机制
3. B+ 树为什么适合数据库索引
4. 聚簇索引、非聚簇索引、回表、覆盖索引
5. 最左前缀、索引失效、慢 SQL 分析
6. 行锁、间隙锁、Next-Key Lock、死锁
7. 乐观锁、悲观锁、Redis 锁和数据库锁的边界

项目关联：

1. 结算状态流转的事务边界
2. ERP 推送失败后的补偿和重跑
3. 幂等控制和数据库唯一约束、状态机的关系

验收标准：

1. 能回答常见 MySQL 追问
2. 能把 MySQL 机制讲到结算系统场景里

---

### Week 3：Redis / MQ / 分布式 + 项目深挖

目标：

1. 把分布式组件从八股答案转成项目表达
2. 整理结算系统 2 到 3 个可深挖亮点

重点任务：

1. Redis 为什么快
2. Redis 持久化、缓存穿透、击穿、雪崩
3. Redis 分布式锁、锁续期、误删、超时释放
4. 缓存和数据库一致性
5. MQ 为什么出现
6. 消息不丢、重复消费、幂等消费、顺序消息、消息积压
7. 最终一致性、补偿、重试、任务分片、多节点并行

项目关联：

1. 云服务结算系统：分布式锁、状态流转、MQ 回写、补偿重跑
2. RAG 系统：异步索引、长耗时任务、失败重试、可观测性

验收标准：

1. 能用 2 分钟讲清结算系统核心链路
2. 能用 10 分钟展开一个一致性或幂等设计
3. Redis / MQ 回答能落到项目，不停留在定义

---

### Week 4：Spring / 并发 / 算法保底 + 模拟面试

目标：

1. 把前 3 周内容转成面试表现
2. 补齐 Spring 和 Java 并发高频点
3. 做至少 2 轮模拟面试复盘

重点任务：

1. IOC、AOP、Bean 生命周期
2. Spring MVC 请求链路、Filter 和 Interceptor
3. `@Transactional` 原理和失效场景
4. `synchronized`、`volatile`、CAS、AQS、线程池
5. 滑动窗口、二分、DP 入门
6. 结算系统和 RAG 项目面试表达

验收标准：

1. 完成至少 2 轮八股 + 项目模拟面试
2. 完成至少 3 道算法代表题的思路复述
3. 形成一份高频错题和表达问题清单

---

## 5. 每天学习节奏

每个 3 到 5 小时学习块建议这样切：

1. 20%：复习上一次遗留问题
2. 40%：学习一个核心机制
3. 20%：结合项目改写表达
4. 20%：回答追问或做小型模拟

每天必须输出：

1. 一个知识点的本质解释
2. 一个面试简洁版回答
3. 一个项目关联表达
4. 一个 mistakes 记录或待复盘问题

---

## 6. 本月判定标准

一个知识点只有满足以下条件，才算本月可用：

1. 能用自己的话解释本质
2. 能说明它解决的真实工程问题
3. 能指出边界和代价
4. 能联系结算系统或 RAG 项目
5. 能给出 1 分钟面试回答
6. 能接住至少 2 个追问

如果只是看懂笔记，状态只能算 `REVIEW`，不能算 `DONE`。
