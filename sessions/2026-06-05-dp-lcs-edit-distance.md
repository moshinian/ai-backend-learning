## 已掌握

### 背包分类

1. 能先判断物品最多一次还是无限次。
2. 能区分可达性、方案数、最小值和最大值。
3. 能从状态定义推出 `dp[0]`。
4. 能解释 0/1 背包倒序防止重复使用。
5. 能解释完全背包正序允许重复使用。
6. 能解释零钱兑换 II 外层遍历硬币是为了避免把组合算成排列。
7. 能把目标和推导为 `P=(sum+target)/2` 的 0/1 背包方案数问题。

### LCS

状态定义：

```text
dp[i][j] = text1 前 i 个字符与 text2 前 j 个字符的最长公共子序列长度
```

转移：

```text
相等：dp[i][j] = dp[i-1][j-1] + 1
不等：dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

已经能解释字符不等时为什么不能只取左上角，以及最终答案为什么是 `dp[m][n]`。

### 编辑距离

状态定义：

```text
dp[i][j] = 将 word1 前 i 个字符转换成 word2 前 j 个字符的最少操作数
```

边界：

```text
dp[i][0] = i
dp[0][j] = j
```

转移：

```text
相等：dp[i][j] = dp[i-1][j-1]
不等：dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1
```

已经能用 `"ab" -> "ac"` 和 `"cat" -> "cut"` 手算二维表。

## 已暴露问题

1. LCS 中一开始混淆 DP 前缀长度和字符串字符下标。
2. 编辑距离中一开始把插入操作也映射到 `dp[i-1][j]`。
3. 容易把插入后的实际字符串长度当成新的 DP 下标。
4. 完整复述时曾把字符相等和不相等的转移分支写反。
5. 手算相邻状态时出现下标抄写错误，结果碰巧正确。
6. 复杂度应写 `O(mn)`，不是统一写成 `O(n²)`，大 O 使用大写。
