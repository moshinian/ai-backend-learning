## 3. 本次暴露的问题

1. 起初认为 Redis 锁基本能保证任务只处理一次，对锁过期和外部副作用边界认识不足。
2. 对 `NX`、`PX`、看门狗等基础概念需要补定义。
3. 最初倾向认为锁一直续期就说明实例没有宕机，但还需要区分续期线程与业务线程健康。
4. 对 Worker 如何感知 Redis 锁丢失一开始没有具体方案。
5. 对外部调用超时后的失败、不确定、成功三种状态需要继续保持边界。
