# DP MISTAKES

## 1. 状态定义没有对齐题目目标

已暴露问题：

- 爬楼梯题目问“有多少种方法”，一开始把 `dp[i]` 定义成“到达第 i 阶的最少步数”。

正确修正：

```text
题目问方法数：dp[i] = 到 i 的方法数
题目问最少代价：dp[i] = 到 i 的最小花费
题目问最大收益：dp[i] = 到 i 的最大收益
```

复盘问题：

> 这道题到底让我求什么？我的 `dp` 是否直接表达了这个目标？

---

## 2. 边界条件没有服从状态定义

已暴露问题：

- 最小花费爬楼梯中，混淆了 `cost[0]`、`dp[0]`、台阶 0、位置 0。
- 一开始在不同状态定义之间切换，导致 `dp[0]` 到底是 `0` 还是 `cost[0]` 不清楚。

正确修正：

推荐定义：

```text
dp[i] = 到达位置 i 的最小花费
```

此时：

```text
dp[0] = 0
dp[1] = 0
dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])
```

注意：

```text
cost[0] 是第 0 个台阶费用
dp[0] 是到达位置 0 的最小花费
```

两者不是一回事。

复盘问题：

> 我的 `dp[i]` 表示的是“踩到台阶 i 后”，还是“到达位置 i”？

## 3. 把 `dp[n-1]` 和 `max(dp)` 混淆

已暴露问题：

- LIS 和最大子数组和中，一开始对最终答案为什么不是 `dp[n-1]` 表达不稳定。

正确修正：

```text
如果 dp[i] = 考虑 0..i 的全局最优，答案通常是 dp[n-1]
如果 dp[i] = 以 nums[i] 结尾的局部最优，答案通常是 max(dp)
```

例子：

```text
打家劫舍：dp[i] = 考虑 0..i 的最大金额，答案 dp[n-1]
LIS：dp[i] = 以 nums[i] 结尾的 LIS 长度，答案 max(dp)
最大子数组和：dp[i] = 以 nums[i] 结尾的最大连续和，答案 max(dp)
```

复盘问题：

> 我的 `dp[i]` 是全局最优，还是带“以 i 结尾”限制的局部最优？

---

## 4. 比较对象说错

已暴露问题：

- LIS 中容易说成 `dp[i] > dp[j]`，但实际判断能否接上应比较 `nums[j] < nums[i]`。

正确修正：

```text
能不能接上：看 nums[j] < nums[i]
接上后的长度：看 dp[j] + 1
```

复盘问题：

> 我现在比较的是原数组的值，还是 DP 状态值？这两个比较分别解决什么问题？

## 5. 用空话解释“不能背公式”

已暴露问题：

- 回答“为什么不能先背公式”时，说成“哲学思想”，不够面试化。

正确表达：

> DP 公式是由状态定义和选择推出来的。题目目标变了，状态定义、边界和聚合方式都会变。如果只背公式，很容易把方法数写成最小值，把 `dp[n-1]` 和 `max(dp)` 搞混，或者让边界条件和状态定义不一致。

复盘问题：

> 这个公式是从哪个状态定义和哪个选择推出来的？

## 6. `dp[0]` 的含义机械记忆

已暴露问题：

- 零钱兑换 II 中一开始认为 `dp[0] = 0`。
- 容易把 `dp[0]` 当成固定值，而不是从状态定义推出。

正确修正：

```text
最少硬币数：dp[0] = 0，表示凑成金额 0 需要 0 枚硬币
组合数：dp[0] = 1，表示凑成金额 0 有 1 个空方案，作为转移种子
可达性：dp[0] = true，表示什么都不选可以凑出 0
```

复盘问题：

> 我的 `dp[0]` 是数量、方案种子，还是可达性边界？

## 7. 把组合数写成排列数

已暴露问题：

- 零钱兑换 II 一开始尝试写成 `dp[i] = dp[i-1] + dp[i-2] + dp[i-5]`。
- 这会按“最后一个硬币是谁”统计，把 `1+2` 和 `2+1` 算成两种。

正确修正：

```text
for coin in coins:
    for i from coin to amount:
        dp[i] += dp[i - coin]
```

原因：

> 外层遍历硬币可以固定硬币种类的引入顺序，避免排列重复。

复盘问题：

> 题目问的是组合数，还是排列数？

## 8. 0/1 背包和完全背包转移来源混淆

已暴露问题：

- 完全背包二维转移中，一开始选择了 `dp[i-1][j-coin]`，这会把当前硬币限制成只能使用一次。

正确修正：

```text
0/1 背包选当前物品：dp[i-1][j-weight]
完全背包选当前物品：dp[i][j-weight]
```

一维压缩：

```text
0/1 背包容量倒序：防止当前物品重复使用
完全背包容量正序：允许当前物品重复使用
```

复盘问题：

> 当前物品只能用一次，还是可以重复使用？

---

## 9. 状态更新时忘记保留旧状态

已暴露问题：

- 完全背包可达性中，处理新硬币时一度认为旧的 `true` 状态会消失。

正确修正：

```text
dp[j] = dp[j] || dp[j - coin]
```

其中：

```text
dp[j] 表示不使用当前 coin 时已经可达
dp[j - coin] 表示使用当前 coin 后可达
```

复盘问题：

> 当前转移里的左边旧 `dp[j]` 表示什么？它是否应该被保留？

---

## 10. 背包分类时只看循环形式

已暴露问题：

- 一度把“每个物品最多选一次，问最大价值”误归为组合数问题。

正确修正：

背包题先看两个维度：

```text
1. 每个物品最多一次，还是每种物品无限次？
2. 题目目标是可达性、方案数、最小值，还是最大值？
```

再决定：

```text
0/1 背包：容量倒序
完全背包：容量正序
组合数：外层物品，避免排列重复
```
