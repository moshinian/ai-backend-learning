# DP BASIC

## 1. 本质

动态规划不是背公式。

它的本质是：

> 把一个大问题拆成会重复出现的小问题，把小问题答案保存下来，再通过状态转移逐步构造最终答案。

学习 DP 时先问四件事：

1. 题目到底要求什么？
2. `dp[i]` 或 `dp[i][j]` 表示什么？
3. 当前状态可以从哪些子问题转移过来？
4. 这些来源应该相加、取 `min`、取 `max`，还是判断可达？

---

## 2. 状态定义必须对齐题目目标

| 题目问法 | 常见状态方向 | 聚合方式 |
|---|---|---|
| 有多少种方法 | `dp[i] = 到 i 的方案数` | 加法 |
| 最少花费 / 最少次数 | `dp[i] = 到 i 的最小代价` | `min` |
| 最大收益 / 最大长度 | `dp[i] = 到 i 的最大收益` | `max` |
| 能不能做到 | `dp[i] = 到 i 是否可达` | 布尔或 |

如果题目问“方法数”，却把 `dp[i]` 定义成“最少步数”，后面的转移一定会错。

---

## 3. 模型一：爬楼梯方法数

题目：

> 每次可以爬 1 阶或 2 阶，爬到第 `n` 阶有多少种方法？

状态定义：

```text
dp[i] = 爬到第 i 阶的方法数
```

最后一步只有两种来源：

| 最后一步 | 子问题 |
|---|---|
| 从 `i-1` 走 1 步 | `dp[i-1]` |
| 从 `i-2` 走 2 步 | `dp[i-2]` |

转移：

```text
dp[i] = dp[i-1] + dp[i-2]
```

边界：

```text
dp[1] = 1
dp[2] = 2
```

也可以引入空方案：

```text
dp[0] = 1
```

这时更方便处理一次可以走 `1、2、3` 阶的变形：

```text
dp[i] = dp[i-1] + dp[i-2] + dp[i-3]
```

---

## 4. 模型二：最小花费爬楼梯

推荐定义：

```text
dp[i] = 到达位置 i 的最小花费
```

注意：

1. `cost` 下标是台阶编号，范围是 `0..n-1`
2. `dp` 下标是位置，范围是 `0..n`
3. 楼顶是位置 `n`，不是台阶 `n`
4. `cost[0]` 是第 0 个台阶费用，不等于 `dp[0]`

边界：

```text
dp[0] = 0
dp[1] = 0
```

原因：

> 题目允许从第 0 阶或第 1 阶开始，到达这两个起点本身不需要花费。

转移：

```text
dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])
```

解释：

> 到达位置 `i`，要么从 `i-1` 来并支付 `cost[i-1]`，要么从 `i-2` 来并支付 `cost[i-2]`，取较小值。

答案：

```text
dp[n]
```

---

## 5. 模型三：打家劫舍

题目：

> 相邻房子不能同时偷，求最大金额。

状态定义：

```text
dp[i] = 考虑下标 0..i 的房子时，能偷到的最大金额
```

当前第 `i` 间房子只有两种选择：

| 选择 | 结果 |
|---|---|
| 不偷第 `i` 间 | `dp[i-1]` |
| 偷第 `i` 间 | `dp[i-2] + nums[i]` |

转移：

```text
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

边界：

```text
dp[0] = nums[0]
dp[1] = max(nums[0], nums[1])
```

答案：

```text
dp[n-1]
```

因为 `dp[i]` 已经表示 `0..i` 范围内的全局最优。

---

## 6. 模型四：最长递增子序列 LIS

题目：

> 求最长严格递增子序列长度。

子序列不要求连续，但相对顺序不能乱。

状态定义：

```text
dp[i] = 以 nums[i] 结尾的最长递增子序列长度
```

为什么要定义成“以 `nums[i]` 结尾”：

> 判断 `nums[i]` 能不能接到前面的序列后面，必须知道前一个序列的结尾值。只保存“前 i 个元素的全局最长长度”会丢失结尾信息。

转移：

```text
如果 j < i 且 nums[j] < nums[i]
dp[i] = max(dp[i], dp[j] + 1)
```

初始化：

```text
dp[i] = 1
```

答案：

```text
max(dp)
```

原因：

> `dp[i]` 是以 `nums[i]` 结尾的局部最优，整体 LIS 不一定以最后一个元素结尾。

复杂度：

```text
时间 O(n^2)
空间 O(n)
```

---

## 7. 模型五：最大子数组和

题目：

> 求最大连续子数组和，子数组不能为空。

状态定义：

```text
dp[i] = 以 nums[i] 结尾的最大连续子数组和
```

转移：

```text
dp[i] = max(dp[i-1] + nums[i], nums[i])
```

解释：

> 以 `i` 结尾的连续子数组，要么接上以 `i-1` 结尾的那一段，要么从 `nums[i]` 重新开始。

答案：

```text
max(dp)
```

原因：

> 最大子数组可能在中间位置结束，不一定以最后一个元素结尾。

全负数边界：

```text
nums = [-3, -2, -5]
answer = -2
```

因为子数组不能为空，答案不是 0。

---

## 8. `dp[n-1]` 还是 `max(dp)`

| 状态定义 | 最终答案 |
|---|---|
| `dp[i] = 考虑 0..i 的全局最优` | 通常是 `dp[n-1]` |
| `dp[i] = 以 nums[i] 结尾的局部最优` | 通常是 `max(dp)` |

例子：

| 题目 | 状态定义 | 答案 |
|---|---|---|
| 打家劫舍 | 考虑 `0..i` 的最大金额 | `dp[n-1]` |
| LIS | 以 `nums[i]` 结尾的 LIS 长度 | `max(dp)` |
| 最大子数组和 | 以 `nums[i]` 结尾的最大连续和 | `max(dp)` |

## 9. 模型六：整数拆分

题目：

> 把整数 `n` 拆成至少两个正整数之和，使乘积最大。

状态定义：

```text
dp[i] = 把整数 i 至少拆成两个正整数后，能得到的最大乘积
```

枚举第一刀：

```text
i = j + (i-j)
```

右边 `i-j` 有两种选择：

| 选择 | 乘积 |
|---|---|
| 不继续拆 | `j * (i-j)` |
| 继续拆 | `j * dp[i-j]` |

转移：

```text
dp[i] = max(dp[i], j * (i-j), j * dp[i-j])
```

其中：

```text
1 <= j < i
```

边界：

```text
dp[2] = 1
```

复杂度：

```text
时间 O(n^2)
空间 O(n)
```

关键点：

> 这题不是固定最后一步，而是枚举第一刀切在哪里，并判断剩余部分拆或不拆。

## 10. 背包 DP 判断框架

背包题先按两个维度判断：

| 维度 | 问题 |
|---|---|
| 物品使用次数 | 每个物品最多一次，还是每种物品无限次？ |
| 题目目标 | 可达性、方案数、最小值、最大值？ |

再决定状态、转移和循环方向。

### 10.1 0/1 背包

每个物品最多用一次。

一维压缩时容量倒序：

```text
for item in items:
    for j from capacity down to weight:
        ...
```

原因：

> 倒序可以保证 `dp[j-weight]` 仍然是上一轮状态，防止当前物品在同一轮被重复使用。

### 10.2 完全背包

每种物品可以无限使用。

一维压缩时容量正序：

```text
for item in items:
    for j from weight to capacity:
        ...
```

原因：

> 正序允许当前物品本轮刚更新出来的状态继续参与后续更新，表示当前物品可以重复使用。

### 10.3 二维状态中的来源差异

0/1 背包：

```text
dp[i][j] = dp[i-1][j] 或 dp[i-1][j-weight]
```

选当前物品时来自上一行，因为当前物品只能用一次。

完全背包：

```text
dp[i][j] = dp[i-1][j] 或 dp[i][j-weight]
```

选当前物品时仍在当前行，因为当前物品可以重复使用。

## 11. 背包模型一：零钱兑换 II

题目：

> 每种硬币无限使用，求凑成 `amount` 的组合数。组合不看顺序。

状态定义：

```text
dp[i] = 凑成金额 i 的组合数
```

边界：

```text
dp[0] = 1
```

含义：

> 凑成金额 0 有一个空方案。它不是实际硬币组合，而是方案数转移的种子。

转移：

```text
for coin in coins:
    for i from coin to amount:
        dp[i] += dp[i - coin]
```

关键点：

1. 外层遍历硬币：固定硬币种类引入顺序，避免把 `1+2` 和 `2+1` 重复统计。
2. 内层金额正序：每种硬币可以无限使用。

---

## 12. 背包模型二：零钱兑换 I

题目：

> 每种硬币无限使用，求凑成 `amount` 的最少硬币数。凑不出返回 `-1`。

状态定义：

```text
dp[i] = 凑成金额 i 所需的最少硬币数
```

边界和初始化：

```text
dp[0] = 0
dp[1..amount] = INF
```

含义：

> 凑成金额 0 需要 0 枚硬币。其他金额先认为不可达。

转移：

```text
dp[i] = min(dp[i], dp[i - coin] + 1)
```

前提：

```text
i - coin >= 0
dp[i - coin] != INF
```

关键点：

> 这题求最少数量，所以不同硬币来源之间取 `min`，不是累加方案数。

---

## 13. 背包模型三：分割等和子集

题目：

> 判断数组能否分成两个和相等的子集。

问题转化：

```text
能否从数组中选出一部分数，使其和等于 sum(nums) / 2
```

如果总和是奇数：

```text
直接返回 false
```

状态定义：

```text
dp[j] = 在已经处理过的数字中，是否可以选出若干个，使和恰好为 j
```

边界：

```text
dp[0] = true
```

转移：

```text
for num in nums:
    for j from target down to num:
        dp[j] = dp[j] || dp[j - num]
```

关键点：

> 这是 0/1 背包可达性问题。每个数字只能用一次，所以容量倒序。

---

## 14. 背包模型四：完全平方数

题目：

> 求整数 `n` 最少可以由多少个完全平方数组成。

状态定义：

```text
dp[i] = 凑成整数 i 所需的最少完全平方数个数
```

边界和初始化：

```text
dp[0] = 0
dp[1..n] = INF
```

可选物品：

```text
1, 4, 9, 16, ...
```

转移：

```text
dp[i] = min(dp[i], dp[i - square] + 1)
```

其中：

```text
square <= i
```

关键点：

> 每个平方数可以重复使用，所以它是完全背包 + 最小值 DP。

## 15. 背包题分类速查

| 题目目标 | 物品使用次数 | 类型 | 一维循环 |
|---|---|---|---|
| 能否凑出 target | 每个物品最多一次 | 0/1 背包 + 可达性 | 容量倒序 |
| 最多能装多少价值 | 每个物品最多一次 | 0/1 背包 + 最大值 | 容量倒序 |
| 凑成 target 的子集数量 | 每个物品最多一次 | 0/1 背包 + 方案数 | 容量倒序 |
| 凑成 amount 的最少硬币数 | 每种硬币无限次 | 完全背包 + 最小值 | 容量正序 |
| 能否凑成 amount | 每种硬币无限次 | 完全背包 + 可达性 | 容量正序 |
| 凑成 amount 的组合数 | 每种硬币无限次 | 完全背包 + 方案数 | 外层硬币，容量正序 |

三句话：

```text
0/1 背包倒序：防止当前物品重复使用。
完全背包正序：允许当前物品重复使用。
组合数外层物品：固定硬币顺序，避免排列重复。
```

### 15.1 `dp[0]` 速查

| 目标 | `dp[0]` | 含义 |
|---|---:|---|
| 最少数量 | `0` | 凑成 0 需要 0 个物品 |
| 最大价值 | `0` | 容量为 0 时价值为 0 |
| 方案数 | `1` | 什么都不选，是 1 个空方案 |
| 可达性 | `true` | 什么都不选，可以凑出 0 |

不要机械背 `dp[0]`，必须从状态定义推出。

### 15.2 分类小测结论

背包分类时按这个顺序说：

```text
1. 0/1 背包还是完全背包
2. 可达性、方案数、最小值还是最大值
3. dp[0] 的含义
4. 容量正序还是倒序
5. 转移用 ||、+=、min 还是 max
```

常见模板：

```text
0/1 背包 + 可达性:
dp[0] = true
for num in nums:
    for j from target down to num:
        dp[j] = dp[j] || dp[j - num]

0/1 背包 + 方案数:
dp[0] = 1
for num in nums:
    for j from target down to num:
        dp[j] += dp[j - num]

0/1 背包 + 最大值:
dp[0..capacity] = 0
for item in items:
    for j from capacity down to weight:
        dp[j] = max(dp[j], dp[j - weight] + value)

完全背包 + 可达性:
dp[0] = true
for coin in coins:
    for j from coin to amount:
        dp[j] = dp[j] || dp[j - coin]

完全背包 + 方案数:
dp[0] = 1
for coin in coins:
    for j from coin to amount:
        dp[j] += dp[j - coin]

完全背包 + 最小值:
dp[0] = 0
dp[1..amount] = INF
for coin in coins:
    for j from coin to amount:
        dp[j] = min(dp[j], dp[j - coin] + 1)
```

## 16. 面试表达模板

简洁版：

> 我做 DP 时会先根据题目目标定义状态。比如求方法数就让 `dp[i]` 表示方案数，求最小花费就表示最小代价，求最大收益就表示最大收益。然后分析最后一步或当前位置的选择，判断不同来源之间是相加、取 `min` 还是取 `max`。最后再根据状态定义确定边界和答案是 `dp[n-1]` 还是 `max(dp)`。

深挖版：

> DP 的关键不是背公式，而是状态定义要同时满足两点：能表示子问题答案，并且保留足够信息支持下一步转移。比如 LIS 如果只定义成前 i 个元素的全局最长长度，就不知道这个序列的结尾值，无法判断当前元素能不能接上；所以要定义成以 `nums[i]` 结尾。状态定义一旦确定，转移、边界和最终答案都必须严格服从这个定义。
