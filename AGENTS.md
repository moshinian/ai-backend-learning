# AGENTS

## 1. 定位

本文件是仓库的文档规则中心。

它只定义：

1. 各类 Markdown 文件的职责边界
2. 各类 Markdown 文件的最小书写格式
3. 内容应该归档到哪里
4. 新增、修改、删除文档的规则
5. 什么时候更新 `START_HERE.md`
6. 什么时候更新 `LEARNING_BACKLOG.md`
7. 什么时候同步 `LEARNING_ROADMAP.md`

它不记录：

1. 个人背景
2. 项目经历
3. 具体学习目标
4. 具体技术知识
5. 面试策略
6. 当前学习任务
7. 具体学习路线

---

## 2. 文档流转规则

| 内容类型 | 写入位置 |
|---|---|
| 长期能力方向 | `LEARNING_ROADMAP.md` |
| 具体可执行学习任务 | `LEARNING_BACKLOG.md` |
| 当前恢复状态 | `START_HERE.md` |
| 单次学习摘要 | `sessions/` |
| 后端技术主笔记 | `backend/` |
| 基础知识主笔记 | `fundamentals/` |
| 项目深挖 | `projects/` |
| 面试表达、真实面试复盘、模拟面试 | `interview/` |
| 求职简历源稿 | `resume/` |
| 错误、误区、纠偏 | `mistakes/` |
| 长期学习画像、阶段性反思、历史索引 | `LEARNING_JOURNAL.md` |
| 可运行学习实验代码 | `labs/` |

不要把同一份内容复制到多个文件。需要互相关联时，只保留简短引用。

当前状态权威顺序：

1. 新会话恢复入口以 `START_HERE.md` 为准。
2. 任务优先级、状态、验收标准、当前断点和下一步动作以 `LEARNING_BACKLOG.md` 为准。
3. `sessions/`、`interview/real-records/` 中的“下次入口 / 后续任务”只表示当次历史归档，不表示当前仍然有效。
4. `backend/`、`fundamentals/`、`projects/`、`interview/`、`resume/`、`mistakes/` 不记录当前任务状态；如需关联任务，只保留指向 `LEARNING_BACKLOG.md` 的简短引用。

## 3. 核心文件最小格式

### `README.md`

必填：

1. 仓库用途
2. 核心文件说明
3. 主要目录职责
4. 新会话恢复方式

边界：

1. 不写当前任务细节
2. 不写长篇学习计划

### `START_HERE.md`

必填：

1. 当前候选任务
2. 当前断点
3. 最近学习位置
4. 下一步动作
5. 优先读取文件

边界：

1. 不写长篇知识内容
2. 不替代 `LEARNING_BACKLOG.md`
3. 不替代 `LEARNING_ROADMAP.md`

### `LEARNING_ROADMAP.md`

必填：

1. 能力模块
2. 模块目标
3. 能力边界
4. 典型沉淀位置

边界：

1. 不写任务状态
2. 不写当前断点
3. 不决定今天具体学什么
4. 不替代 `LEARNING_BACKLOG.md`

### `LEARNING_BACKLOG.md`

必填：

1. ID
2. 优先级
3. 来源
4. RoadmapRef
5. 状态
6. 主题
7. 学习目标
8. 验收标准
9. 当前断点
10. 关联文件
11. 下一步动作

边界：

1. 只记录任务
2. 不写成长篇知识笔记
3. 不保存单次学习过程

### `LEARNING_JOURNAL.md`

必填：

1. 长期学习画像
2. 阶段性反思
3. 长期误区
4. 重要变化
5. 历史索引

边界：

1. 不做任务调度
2. 不替代 `START_HERE.md`
3. 不替代 `LEARNING_BACKLOG.md`

## 4. 目录最小格式

### `sessions/`

必填：

1. 日期
2. 主题
3. 本次做了什么
4. 关键结论
5. 下次入口
6. 关联文件

边界：

1. 只做单次学习摘要
2. 不作为新会话主恢复入口

### `mistakes/`

必填：

1. 错误表现
2. 根因
3. 正确理解
4. 复盘触发条件
5. 关联主题

边界：

1. 只记录错误、误区和纠偏
2. 不写完整知识笔记

### `interview/`

必填：

1. 问题
2. 现场回答或目标回答
3. 评价
4. 改进版
5. 后续任务

边界：

1. 保存面试表达、真实面试复盘、模拟面试
2. 不写主知识笔记

### `backend/`

必填：

1. 本质
2. 解决的问题
3. 机制
4. 边界
5. 项目或工程关联
6. 面试表达入口

边界：

1. 写后端技术正式沉淀
2. 不记录当前任务调度

### `fundamentals/`

必填：

1. 本质
2. 解决的问题
3. 机制
4. 边界
5. 工程关联
6. 面试表达入口

边界：

1. 写基础知识正式沉淀
2. 不记录当前任务调度

### `projects/`

必填：

1. 项目背景
2. 解决的问题
3. 核心链路
4. 关键设计
5. 边界和风险
6. 面试表达入口

边界：

1. 写项目深挖
2. 不替代真实面试复盘

### `resume/`

必填：

1. 求职意向
2. 个人优势
3. 专业技能
4. 工作经历
5. 项目经历
6. 教育经历

边界：

1. 只保存可维护的求职简历源稿和目录说明
2. Java 后端和 AI 应用简历是平行有效的岗位版本，不在文件职责上设置主次
3. 两份简历共享的工作时间、项目事实和量化数据必须保持一致
4. 不记录当前投递状态、学习任务或面试复盘
5. 不把未落地方案、学习实验或待验证数据写成生产经历
6. 版本控制中的文件不保存手机号、私人邮箱、详细住址等敏感信息，使用占位符代替

### `labs/`

必填：

1. 实验目标
2. 运行方式
3. 关键验证点
4. 边界说明
5. 关联任务或主题

边界：

1. 只保存用于验证机制的可运行学习实验代码
2. 不替代正式项目深挖
3. 不替代主题知识笔记
4. 不记录当前任务调度

## 5. Backlog 规则

优先级：

1. P0：当前最想学、最有动力、最应该优先推进的知识
2. P1：面试、项目表达、实际问题暴露出的短板
3. P2：长期重要但不紧急的基础知识

状态：

1. TODO：待学习
2. DOING：正在学习
3. REVIEW：已学完一轮，等待复盘或表达验证
4. DONE：已完成当前验收标准
5. BLOCKED：暂时卡住或等待外部信息

任务来源：

1. `LEARNING_ROADMAP.md`
2. 真实面试暴露
3. 项目表达短板
4. 临时技术问题
5. 个人主动学习
6. 长期基础补课
7. 旧冲刺计划迁移

Roadmap 关系：

1. 来自 Roadmap 的任务必须填写 `RoadmapRef`
2. 有长期价值但 Roadmap 缺节点时，先补 Roadmap 能力节点，再加入 Backlog
3. 临时问题可以只进入 Backlog
4. 临时问题反复出现或被确认有长期能力价值时，需要回补 Roadmap

任务暂停时必须更新：

1. 状态
2. 当前断点
3. 下一步动作
4. 关联文件

任务完成时必须更新：

1. 状态
2. 验收结果
3. 关联沉淀文件
4. 是否需要回补 `LEARNING_ROADMAP.md` 或 `mistakes/`

## 6. 更新触发规则

更新 `START_HERE.md`：

1. 当前候选最高优先级任务变化
2. 当前断点变化
3. 下一次新会话需要读取的文件变化

更新 `LEARNING_BACKLOG.md`：

1. 新增可执行学习任务
2. 任务暂停
3. 任务完成
4. 任务优先级变化
5. 任务验收标准变化

更新 `LEARNING_ROADMAP.md`：

1. 新增长期能力方向
2. 调整长期能力模块边界
3. 临时问题反复出现并确认具有长期建设价值

更新 `LEARNING_JOURNAL.md`：

1. 出现长期学习方式变化
2. 出现阶段性反思
3. 出现反复发生的长期误区
4. 需要补充历史索引

---

## 7. 修改文档前后要求

修改前：

1. 先读取相关现有文件
2. 确认内容应该归档到哪里
3. 避免把同一内容写入多个位置
4. 涉及删除或大范围重写时，先说明计划

修改中：

1. 优先小范围修改
2. 不大规模改写既有知识笔记
3. 不把任务调度写入主知识笔记
4. 不把具体学习内容写入 `AGENTS.md`

修改后：

1. 检查核心文件职责是否清晰
2. 检查是否产生失效引用
3. 必要时更新 `START_HERE.md`
4. 必要时更新 `LEARNING_BACKLOG.md`
