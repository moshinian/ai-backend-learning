# RESUME

## 1. 目录用途

本目录保存可维护的求职简历 Markdown 源稿。

当前规划的岗位版本：

1. `java-backend-resume.md`
2. `ai-application-resume.md`

两份简历平行有效，不在文件职责上设置主次。实际投递比例和当前求职安排不在本目录记录。

## 2. 事实边界

1. 两份简历可以针对目标岗位调整内容顺序和表达重点。
2. 工作时间、教育经历、项目事实、个人职责和量化数据必须保持一致。
3. 不把学习实验、未落地设计或待验证方案包装成生产经历。
4. 对事实存在疑问时，先保留原文或标记待确认，不自行补充。

## 3. 隐私规则

1. Git 中的 Markdown 源稿使用 `[手机号]`、`[邮箱]` 等占位符。
2. 不保存身份证号、详细住址等敏感信息。
3. 带真实联系方式的最终 PDF / Word 投递文件不进入版本控制。

## 4. 当前文件

1. `ai-application-resume.md`：此前持续投递使用的 AI 应用工程师简历原文脱敏归档。
2. `java-backend-resume.md`：面向 Java 后端工程师岗位的平行有效简历版本。
