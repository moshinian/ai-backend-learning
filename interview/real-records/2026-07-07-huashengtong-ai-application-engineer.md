### 问题 7：结果融合与 Rerank

**面试官原问题**

> 双路召回结果如何融合？Rerank 模型内部机制是什么？

**我的现场回答**

> 采用归一化权重法，基于评测指标确定权重；补充可以用特定网络重排，但对重排模型作用和内部机制说明不清，泛化为语料训练、编码器-解码器和向量转换。

**评价**

- 评分：5/10
- 回答正确的部分：知道融合需要统一量纲，可以通过评测指标调权重。
- 主要问题：Rerank 机制没有讲到点上。面试官想听 query-document pair 交互打分、cross-encoder、相关性标签、hard negative、私有语料微调等。

**更好的回答**

> 第一阶段召回常用 bi-encoder，把 query 和 chunk 分别编码成向量，适合快速召回。Rerank 通常更像 cross-encoder，把 query 和候选 chunk 拼在一起输入模型，让模型直接判断这一对是否相关，输出相关性分数。它比向量召回慢，但能利用 query 和文本之间更细粒度的 token 交互，所以适合对 TopK 候选做精排。私有语料场景下，如果通用 reranker 不适配，需要构造业务查询、正例片段和 hard negative，微调 reranker 或加入业务特征做融合。

**需要补充学习**

1. Bi-encoder vs cross-encoder。
2. Rerank 训练数据、hard negative、私有语料微调。
3. RRF、归一化加权、rerank 的边界。

## 5. 跨题错误模式

1. RAG 工程链路能讲，但 Rerank / NLP 算法机制一被深挖就容易泛化。
2. 文档解析回答偏“文本处理”，对 Word / PDF / OCR / 表格 / 图片等复杂文档结构准备不足。
3. 面试中能诚实划边界，但如果边界后没有补一层“我理解下一步怎么补”，会显得短板只是短板。
4. 工程方案中出现“丢弃旧量”等表达时，需要先判断数据是否允许丢弃，金融场景尤其要谨慎。
5. 前沿算法关注不能只说“看推文”，要形成和岗位相关的持续跟踪方法。
