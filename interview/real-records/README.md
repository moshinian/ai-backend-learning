# 真实面试记录

这个目录用于保存参加真实面试后的完整复盘。

## 1. 与其他目录的边界

| 目录 | 保存内容 |
|---|---|
| `interview/real-records/` | 真实面试原题、现场回答、逐题评价、修正版和后续任务 |
| `interview/mock-records/` | 模拟面试过程、评分和训练结果 |
| `sessions/` | 学习过程摘要和下次恢复断点 |
| `mistakes/interview/` | 跨多次面试反复出现的错误模式 |
| `interview/*-questions.md` | 按技术主题整理后的稳定面试答案 |

## 2. 文件命名

```text
YYYY-MM-DD-岗位简称.md
```

例如：

```text
2026-06-10-llm-application-engineer.md
```

日期使用实际面试日期，不使用复盘日期。

## 3. 记录原则

1. 先还原面试官原问题和自己的现场回答，不要直接改成标准答案。
2. 明确区分事实记忆、复盘推断和后续补充。
3. 每道题记录评价、主要问题和更好的回答方式。
4. 不虚构没有做过的项目能力。
5. 面试结束后提炼跨题错误，必要时同步到 `mistakes/interview/`。
6. 只有多次面试稳定暴露同一方向时，才考虑调整长期 Roadmap。

## 4. 使用模板

新面试复盘复制并填写：

`interview/real-records/TEMPLATE.md`
